# Travel Planner Site Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 构建并发布一个无需自建服务器、数据保存在用户浏览器中的旅行规划网页，支持多行程、时间线、全球地图、消费与多人 AA、出发前行李清单和文件备份。

**Architecture:** 使用 Sites 标准 React/Vinext 项目作为应用外壳，全部业务逻辑在浏览器执行；IndexedDB 是行程数据的唯一持久化来源，导入/导出 JSON 是跨设备迁移和备份方式。地图用 MapLibre GL JS 渲染 OpenStreetMap 衍生矢量底图，业务坐标统一为 WGS84，不依赖后端、登录、云同步、自动地理编码或自动路线服务。

**Tech Stack:** TypeScript、React、Vinext/Sites、IndexedDB + Dexie、Zod、MapLibre GL JS、GeoJSON、Vitest、React Testing Library、fake-indexeddb。

---

## 1. 已确认范围

### 1.1 V1 必须交付

- 创建、编辑、复制、归档和删除多个行程。
- 按日期组织多个行程节点，记录地点、时间、内容与备注。
- 设置节点之间的交通方式、起止时间、班次和费用。
- 地图支持中国大陆和海外地点，提供全程全览与单日详情。
- 消费可在行程前、途中或结束后随时添加，并区分预计、已支付、已取消。
- 支持平均、指定成员、份数、比例和固定金额 AA。
- 行李是出发前准备清单，不包含负责人。
- 数据仅保存在本机浏览器，支持完整 JSON 导入、导出与备份提醒。
- 支持桌面端、手机端、打印行程和无鼠标操作。
- 通过 Sites 发布，不要求用户配置或维护服务器。

### 1.2 V1 明确不做

- 登录、账号、云同步、多人实时协作和修改冲突合并。
- 酒店机票预订、支付、照片与附件上传。
- AI 自动生成行程、实时汇率和实时导航。
- 自动地址补全、自动道路算路、公共 Nominatim 高频调用。
- 高德、百度等 GCJ-02/BD-09 地图源混用。

### 1.3 成功指标

- 用户首次进入后 3 分钟内能建立一个包含 3 个地点的行程。
- 刷新页面、关闭浏览器再打开后，行程仍存在。
- 一个包含 30 天、300 个节点、500 条消费和 200 件行李的行程仍可流畅编辑。
- AA 计算满足金额守恒，舍入误差不超过最小货币单位。
- 导出文件重新导入后，业务数据语义等价。
- 北京、上海、东京、巴黎、纽约等测试坐标均能正确显示在同一地图中。
- 手机宽度 360px 下不存在横向滚动，核心操作可触达。

## 3. 核心数据模型

```ts
export type CurrencyCode = string;

export interface Trip {
  id: string;
  schemaVersion: 1;
  title: string;
  startDate: string;
  endDate: string;
  timezone: string;
  defaultCurrency: CurrencyCode;
  participantIds: string[];
  archivedAt?: string;
  createdAt: string;
  updatedAt: string;
}

export interface Participant {
  id: string;
  tripId: string;
  name: string;
}

export interface Stop {
  id: string;
  tripId: string;
  date: string;
  sortOrder: number;
  title: string;
  country?: string;
  city?: string;
  latitude: number;
  longitude: number;
  startsAt?: string;
  endsAt?: string;
  content?: string;
  notes?: string;
}

export type TransportMode =
  | "walk" | "bike" | "bus" | "metro" | "taxi" | "drive"
  | "train" | "highSpeedRail" | "flight" | "ferry" | "other";

export interface Leg {
  id: string;
  tripId: string;
  fromStopId: string;
  toStopId: string;
  mode: TransportMode;
  departsAt?: string;
  arrivesAt?: string;
  serviceNumber?: string;
  notes?: string;
  routeGeoJson?: GeoJSON.LineString;
  expenseId?: string;
}

export type ExpenseStatus = "planned" | "paid" | "cancelled";
export type SplitMethod = "equal" | "shares" | "percentage" | "fixed";

export interface Expense {
  id: string;
  tripId: string;
  title: string;
  amountMinor: number;
  currency: CurrencyCode;
  status: ExpenseStatus;
  category: string;
  payerParticipantId?: string;
  beneficiaryParticipantIds: string[];
  splitMethod: SplitMethod;
  splitValues: Record<string, number>;
  occurredAt?: string;
  stopId?: string;
  legId?: string;
  notes?: string;
  createdAt: string;
  updatedAt: string;
}

export interface PackingItem {
  id: string;
  tripId: string;
  category: string;
  title: string;
  quantity: number;
  required: boolean;
  packed: boolean;
  notes?: string;
  sortOrder: number;
}
```

金额统一保存为最小货币单位整数，例如 CNY 12.34 保存为 `1234`，避免浮点误差。不同币种分别统计与结算；V1 不自动换汇。

## 6. 主要风险与控制

| 风险 | 影响 | 控制措施 |
|---|---|---|
| 用户清理浏览器数据 | 行程丢失 | 显眼的导出入口、7 天备份提醒、导入恢复 |
| 免费底图在部分网络访问不稳定 | 地图空白或加载慢 | 地图源集中配置、失败提示、业务内容不依赖地图加载成功 |
| 中国大陆坐标系混用 | 标记偏移 | 全链路 WGS84，不接入 GCJ-02/BD-09 底图 |
| AA 小数舍入 | 合计不一致 | 最小货币单位整数、稳定余数分配、金额守恒测试 |
| 大行程导致渲染变慢 | 编辑体验下降 | 按日渲染、列表分段、地图图层最小化、性能 fixture |
| 导入恶意或损坏文件 | 崩溃或污染数据 | Zod 全量校验、大小限制、事务写入、未知版本拒绝 |
| 地图服务政策变化 | 底图失效 | provider 适配层、署名、发布前复核服务条款 |
